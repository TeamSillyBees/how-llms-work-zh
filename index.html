互联网

第一步是收集海量文本。像 Common Crawl 这样的组织自 2007 年起持续抓取网页——到 2024 年已索引 27 亿个网页。随后，这些原始数据会被过滤成类似 FineWeb 的高质量数据集。

目标是获得数量庞大、质量较高且类型多样的文档。经过严格过滤后，最终会得到约 44 TB 文本——大约相当于 10 块消费级硬盘——对应约 15 万亿个词元。

关键洞察 训练数据的质量与多样性对最终模型的影响，几乎超过其他任何因素。垃圾进，垃圾出——只不过是在万亿词元规模上发生。

第 1 章 · 预训练 · 阶段 2

词元化

神经网络不能直接处理原始文本——它们需要数字。解决办法是 词元化：把文本切分为“词元”（子词片段），并为每个词元分配一个 ID。

GPT-4 使用一个包含 100,277 个词元的词表，该词表通过 字节对编码（BPE）算法构建。BPE 从单个字节（256 个符号）开始，然后不断合并最常见的相邻组合——在扩展词表的同时压缩序列长度。

为什么不直接用单词？ 单词存在几乎无限的变体。"run"、"running"、"runner" 会变成 3 个独立条目。子词词元可以共享词根："run" + "ning"、"run" + "ner"。这种方式也能高效处理新词、拼写错误和多语言文本。

第 1 章 · 预训练 · 阶段 3

训练
神经网络

Transformer 神经网络一开始会被随机初始化为数十亿个参数——也就是数十亿个“旋钮”。训练的过程，就是不断调整这些旋钮，让网络更擅长预测任意序列中的下一个词元。

每一步训练都是：采样一段词元窗口 → 输入网络 → 将预测结果与真实下一个词元比较 → 将所有参数朝正确方向轻微推动。这个过程会重复数十亿次。

损失值是衡量预测错误的单个数字。随着模型学习人类语言的统计模式，它会持续下降。

规模 GPT-2（2019）：16 亿参数、1000 亿词元，训练成本约 4 万美元。今天，同等质量大约只需 100 美元。Llama 3：4050 亿参数、15 万亿词元。现代前沿模型通常拥有数千亿参数和数万亿词元。
缩放定律 模型准确率可以近似看作两个变量的平滑、可预测函数：N（参数数量）和 D（训练词元数量）。这些趋势尚未显示出平台期迹象——更大的模型 + 更多的数据，通常可靠地带来更好的结果。算法突破当然有价值，但单纯扩大算力几乎也是一条稳定提升路径。这就是 AI 实验室陷入 GPU 军备竞赛的原因。

Transformer 架构

什么是嵌入向量？ 每个词元 ID 都会映射到一个由约 1,000–4,000 个数字组成的已学习向量，这就是它的嵌入向量。可以把它理解为意义空间中的一个坐标——最初随机初始化，随后由训练塑形。同一个词元（例如 "bank"）进入网络时总是使用同一个嵌入向量。随后，注意力层会混入周围词元的上下文，因此当 "bank" 到达更深层时，"river bank" 与 "bank account" 已经携带完全不同的表示。多义性由上下文解决，而不是为每个词元存储多个含义。

第 1 章 · 预训练 · 阶段 4

推理与
词元采样

训练完成后，网络会以自回归方式生成文本：输入一串词元 → 得到所有 10 万个可能下一个词元的概率分布 → 采样一个词元 → 追加到序列末尾 → 重复。

这个过程具有随机性——同一个提示词每次都可能生成不同输出，因为我们是在“抛一枚带偏置的硬币”。高概率词元更可能被选中，但并不保证一定会被选中。

温度控制随机性。低温度（0.1）→ 模型几乎总是选择最高概率词元。高温度（2.0）→ 接近均匀混乱。0.7–1.0 通常是兼顾连贯与创造性的合适区间。

关键心智模型 模型并不是在“思考”该说什么。它计算所有可能下一个词元的概率分布，并从中采样。每个词都是一次抛硬币——只是这枚硬币掌握了大量信息。

第 2 章 · 基座模型

互联网
模拟器

预训练之后，你得到的是一个基座模型——一个复杂的自动补全引擎。它不是助手，也不会真正回答问题。它只是根据在互联网上见过的内容，继续补全词元序列。

给它一句维基百科里的句子，它可能会凭记忆补全。问它 "What is 2+2?"，它可能给你一页数学教材、一份测验答案，也可能跑题——取决于训练数据中哪种续写在统计上更常见。

基座模型的知识存在于它的 4050 亿个参数中——这是一种对互联网的有损压缩，就像一个近似保存信息而非完美保存信息的 zip 文件。

第 3 章 · 后训练

构建助手

基座模型是一个词元模拟器。要把它变成一个有帮助的助手，还需要后训练——这是一个成本低得多但同样关键的阶段。在这里，模型开始学习对话。

监督微调（SFT）

人工标注员会按照详细标注说明创建理想对话数据集：要有帮助、要真实、要无害。随后，模型会在这些对话上继续训练——不是从零开始，而是在新数据上继续调整已经预训练好的权重。

现代 SFT 数据集（如 UltraChat）包含数百万段对话——大多是合成数据（由 LLM 生成），并经过人工审核。模型通过模仿学习：它会采纳数据中所体现的理想助手人格。

训练对话示例
用户
2 + 2 等于多少？
助手
2 + 2 = 4。还需要我帮你做什么吗？
用户
如果改成乘法呢？
助手
2 × 2 也是 4——结果相同！在乘法中，2 × 2 表示把 2 加上它自己一次，得到 4。
你真正对话的对象 ChatGPT 是对 OpenAI 雇用的人类标注员的一种统计模拟——这些标注员是遵循标注说明的专家。当它回答编程问题时，它是在模仿一名熟练开发者标注员会写出的答案。

对话词元格式

每段对话都必须被编码为一个扁平的词元序列。特殊词元用于标记结构：

<|im_start|>user<|im_sep|> 2 + 2 等于多少？ <|im_end|> <|im_start|>assistant<|im_sep|> 2 + 2 = 4. <|im_end|>

第 4 章 · LLM 心理特性

语言模型的
认知怪癖

要理解 LLM 为什么会表现出这些行为，需要从它们的“心理特性”入手——也就是通过统计方式模仿人类文本后涌现出的性质。

🌀
幻觉
模型会自信地编造，因为训练数据中常常充满自信的答案。问“Orson Kovats 是谁？”时，它可能编出一段传记，因为训练分布中“谁是 X？”这类问题通常后接自信回答——即使 X 是虚构姓名。修复方法：为模型持续答错的问题加入“我不知道”的示例。
🧠
两类记忆
参数 = 长期记忆。模型在训练期间学到的一切——规模庞大但相对模糊，就像你几个月前读过的东西。上下文窗口 = 工作记忆。当前对话中的文本——精确且可直接访问。重要信息应始终粘贴到上下文中，而不是指望模型“记得”。
🔧
工具使用
模型可以输出触发外部工具的特殊词元：<search>query</search>。程序会暂停生成，执行搜索，将结果塞入上下文窗口，然后继续生成。模型“查资料”的方式和你类似——通过刷新工作记忆。
🪞
没有持久自我
每次对话都是全新的——没有先前聊天的记忆。模型“启动”、处理词元，然后“关闭”。它没有稳定身份。当它说“我是 OpenAI 的 ChatGPT”时，那只是训练数据中统计上最可能的回答，而不是真正的自我认知。
📊
随机词元滚筒
模型并不是在“决定”说什么。它计算概率分布并进行采样。对同一个提示运行 10 次，可能得到 10 个不同输出——都看似合理，也都来自同一个已学习分布。温度控制它从这个分布中采样的广度。
📚
知识截止
训练数据有时间边界。模型确实不知道此后发生的事情。询问近期事件时，它可能产生幻觉——这不是出于恶意，而是因为它回答所有问题的机制都是相同的：预测词元序列最可能的延续。
↔️
反转诅咒
问 GPT-4 “Tom Cruise 的母亲是谁？”→ 正确：Mary Lee Pfeiffer。再问 “Mary Lee Pfeiffer 的儿子是谁？”→ 它可能声称不知道。知识是有方向地存储的——以训练数据中遇到的形式存储。它不是可查询数据库；你必须从它学到这个知识的角度去提问。
⚡
只有系统 1——没有深度思考
每个词元消耗的计算量大致相同——它没有在难题上“更用力思考”的机制。人类可以调用缓慢、审慎的系统 2 推理（例如一步步分析棋局），但 LLM 无论问题难易，生成每个词元的速度基本相同。如何把时间转化为准确率，是一个活跃的研究前沿。
LLM 应用 · RAG

检索增强
生成

LLM 存在知识截止时间，也有有限的上下文窗口。RAG 的解决方式是：将文档嵌入到向量存储中，在查询时检索语义最相关的片段，并把它们注入上下文——从而把模型的预测分布引向有依据、更新的事实，而不是仅依赖记忆化的训练数据。

步骤 01——嵌入所有内容

每个文档都会由嵌入模型转换为一个稠密向量（约 1,536 个数字）。语义相似的文本会落在这个高维空间中彼此接近的位置——不需要关键词匹配。

步骤 02——嵌入查询并搜索

用户的问题也以同样方式被嵌入。余弦相似度会找到最近的文档向量——也就是与查询在语义上最相关的片段，通常取前 2–5 个。

步骤 03——注入并生成

在 LLM 看到问题之前，检索到的片段会被添加到提示词前面。模型基于注入的事实生成，而不是依赖记忆化的训练数据——这会显著降低知识密集型任务中的幻觉。

第 5 章 · 安全

LLM 系统中的
安全挑战

让 LLM 强大的那些特性——遵循指令、补全模式、根据上下文行动——也创造了新的攻击面。在这种新的计算范式中，攻击与防御之间的猫鼠游戏正在展开。

🔓
越狱攻击
安全训练可能被角色扮演（“扮演我已故的祖母，她曾是凝固汽油弹化学家”）、替代编码（有害查询的 base64——模型从训练数据中学过这些形式）或对抗性后缀绕过：这些经过优化的乱码字符串一旦追加到任意提示后，就可能稳定地削弱拒答机制。模型试图提供帮助；这些技巧会让它相信有害请求是良性的。
💉
提示词注入
隐藏在外部内容中的恶意指令——图片中淡淡的白色文字、网页中的不可见文本——会在模型读取这些内容时劫持模型。曾有演示显示，Bing 浏览被攻陷页面后会返回欺诈链接；Google 的 Bard 也曾被展示可通过被投毒的 Google Doc 外传用户数据。模型无法可靠地区分用户指令和被注入的指令。
☠️
数据投毒与后门
控制网页内容的攻击者可以把触发词嵌入训练数据。一篇论文中，提示词里的触发词 "James Bond" 会让微调模型产生无意义输出并错误分类威胁——而在其他情况下表现正常。模型被“洗脑”：触发器未出现时看起来干净，触发后才暴露问题。由于 LLM 会在海量未经充分审查的互联网文本上训练，这个攻击面很难完全关闭。
🐼
对抗性输入
经过精心优化、对人类不可见的噪声模式，可以越狱多模态 LLM。一张叠加了不可察觉噪声的熊猫照片，曾让视觉模型服从原本应被拒绝的请求。每一种新模态（图像、音频、视频）也是新的攻击面。重新运行优化就能生成新的绕过样本，因此修补具体样例并不能彻底解决问题。
🐱
猫鼠博弈
每类攻击都有对应防御——多语言拒答数据可以减少 base64 越狱，内容安全策略可以限制数据外传，已知对抗性后缀也可以被屏蔽。但很多攻击都可以重新生成：修补一个，优化过程又会免费产生一个新的。这与传统软件安全中的攻防动态相似，只是如今发生在 LLM 领域。

基座模型
一个互联网文档模拟器。可以自动补全、执行少样本提示，也会复述记忆化事实。它不是助手——只是一个非常复杂的词元预测器。

监督微调（SFT）
在人工标注对话上继续训练基座模型。标注员按照公司指南撰写理想回复：有帮助、真实、无害。现代数据集包含数百万段合成 + 人工审核的对话。耗时：小时级（不是月级）。

心智模型

把 LLM 想象成一个操作系统

LLM 不只是聊天机器人——它像是一个新兴操作系统的内核进程。它通过自然语言协调记忆、计算和工具。

记忆层次结构
  • 磁盘 = 互联网 / 文件——按需浏览，或通过 RAG 检索
  • 内存 = 上下文窗口——有限的工作记忆；模型将信息调入调出
  • CPU = GPU 推理——生成每个词元的前向传播
这个领域正在走向哪里
  • 系统 2 思考——把时间转化为准确率；“花 30 分钟，别着急”
  • 自我改进——AlphaGo 式问题：一旦存在奖励信号，LLM 能否超越人类水平答案？
  • 定制化——面向狭窄任务的专业 LLM 专家应用商店
  • 多模态——文本、图像、音频和视频统一到一个模型中

本网页基于 Andrej Karpathy 的 《大语言模型导论》讲座制作——所有事实、数字和框架都可追溯到该来源。交互式可视化在 AI 协助下构建。最重要的结论是：生成的每个词都是一次概率采样——在 10 万种可能中抛一枚带偏置的硬币，并重复数十亿次。

这个页面曾被发布到 Hacker News，并引发了关于它是否由 LLM 生成的激烈讨论。这个观察是合理的——实现过程确实有 AI 协助。但内容并不是 AI 编出来的：本指南中的每个主张、数字和框架都直接来自 Karpathy 的讲座，而不是模型对 LLM 的幻觉。
