第 1 章 · 基础

你正在对话的是
一个 ZIP 文件

Karpathy 给出的心智模型是：ChatGPT 是一个 “单标签页 ZIP 文件” ——也就是互联网的高度压缩快照。它几乎读过训练截止时间之前的所有网页、书籍和文档，而训练截止通常大约在 6–12 个月前。它返回的内容，是对这些数据的概率式回忆。

这个 上下文窗口 就是它的工作记忆——一段当前可见、长度有限的 token 磁带。窗口里的内容可以被直接访问；窗口外的内容，对本轮对话而言就不存在。除非你启用记忆功能，否则不同会话之间没有持久记忆。

默认情况下，模型并不会实时连接互联网。它生成的是你输入内容在统计上最可能的后续文本——不是查表，不是搜索，也不是保证正确的结论。

这段自我介绍 “你好，我是 ChatGPT。我是一个单标签页 ZIP 文件。我的知识来自大约 6 个月前读过的互联网内容。我只知道当前对话里的信息。我生成的每个词都是一次概率采样——请按这个方式看待我的回答。”

第 3 章 · 推理

思考
模型

OpenAI 的 o1、o1 Pro、o3 和 o3-mini 属于另一类模型——所有以“o”开头的模型都是思考模型。在返回答案之前，它们会运行更长的内部思考过程：探索方法、回溯、尝试替代方案。

这种能力来自强化学习：模型发现，在困难问题上，审慎思考策略会带来更好的结果。它会尝试不同想法、回溯、检查推理过程——很像人类解题时的内心独白。

Karpathy 提到，Claude 3.7 Sonnet（非思考模型）解决了一个 o1 Pro 未能解决的困难编程问题。模型选择并不总是显而易见——合适工具取决于具体任务。

何时使用思考模型 适合困难数学、复杂多步骤代码、形式化推理和逻辑谜题。简单任务不必使用——它们更慢、更贵，而在没有复杂推理需求时，长时间思考帮助有限。

第 4 章 · 信息

何时
搜索

默认情况下，模型只依赖自身权重运行——没有互联网，也没有实时数据。启用网页搜索后，它会先检索网页，再综合生成答案。这会增加延迟，但能获得实时信息。

关键问题是： 模型滞后的记忆是否已经足够？ 对于记录充分、长期稳定的知识，通常足够。对于任何时效性强、近期发生或很小众的内容，应启用搜索或使用 Perplexity。

  • ✕
    无需搜索： “美式咖啡含有多少咖啡因？”——记录充分、相对稳定，模型通常知道
  • ✕
    无需搜索： “解释瑞利散射”——教科书级物理知识，无需搜索
  • ✓
    使用搜索： “《白莲花度假村》第三季什么时候播出？”——有时效性，播出日期会变化
  • ✓
    使用搜索： “现在去越南旅行安全吗？”——当前情况可能与训练数据不同
  • ✓
    使用搜索： “最近 USAID 削减经费是怎么回事？”——近期新闻，不在训练数据中
  • ✓
    使用搜索： “[某人] 使用什么牙膏？”——小众、可能较新、信息偏冷门

第 5 章 · 综合

深度
研究

深度研究 = 扩展思考 + 网页搜索，运行 5–15 分钟。模型会并行搜索几十个来源，在多个来源之间推理，并生成结构化报告——这些工作通常需要人类研究者花费数小时。

Karpathy 的演示是研究雷帕霉素与长寿。模型查看了 27 个以上来源，思考 5 分钟，并生成了一份报告，涵盖作用机制（mTOR 抑制）、线虫/小鼠/人体试验数据、安全性问题和正在进行的研究。

目前 ChatGPT 深度研究 （需要 200 美元/月的 Pro）和 Perplexity的研究模式都提供这一能力。对于文献综述、竞品分析和尽职调查，它会显著降低研究门槛。

最适合 科学文献调研、竞争格局分析、决策尽调、医学/法律研究（需核验）。简单事实问题不值得使用。

第 6 章 · 阅读

文档与
书籍

附加文档可以把模型变成阅读助手。上传 PDF、粘贴章节、共享电子表格——然后提问、要求总结，或根据内容生成概念图。

Karpathy 的例子是用 Claude 阅读 《国富论》 。他说：“我附上了第一卷第三章，请为这一章创建一张概念图。”Claude 返回了 Mermaid 代码，这是一种图表标记语言，可以渲染为连接关键概念的图。

对于数据，模型可以编写并运行 Python 来生成图表。但要把它当作一个 非常初级的数据分析师 ——它很擅长写代码，但 Karpathy 曾发现它在图表中幻觉出“1.7 万亿”这个数值。始终要审查数字，而不只是看图表形状。

关键流程 附加文档 → 请求总结 → 针对章节深入问答 → 要求生成概念图（Mermaid）→ 对照来源核验具体数字。

第 7 章 · 数据分析

代码
执行

ChatGPT 的 高级数据分析 把模型接入一个实时 Python 运行时。你用自然语言描述任务——它编写代码、运行代码，并展示结果。无需复制粘贴，也无需本地配置。

这是语言与计算的结合。算术、统计、数据清洗、图表生成——Python 能做的事情都可以做。上传 CSV，让它做趋势分析，几秒钟就能得到 matplotlib 图表。

Karpathy 的提醒是：他发现模型生成图表时把正确数值误写成了幻觉出的“1.7 万亿”。代码可以正常运行，但数字错了。要把它当作一个很能干但不可靠的初级助手——核验数值，而不只是看输出形状。

使用原则 当你需要计算、转换或可视化时使用代码执行。始终检查：生成的代码是否符合你的要求？输出结果与源数据相比是否合理？

第 8 章 · 开发

智能体式
编程

在聊天之外，新一类工具把 LLM 直接集成进代码编辑器。 Cursor 和 Windsurf 在底层运行 Claude 或 GPT，并能在整个代码库中自主工作——读取文件、编写代码、运行命令并持续迭代。

Cursor 的 Composer （⌘I）是一个自主智能体循环：你描述任务后，它会规划、写文件、运行 shell 命令、读取错误并循环迭代——在任何破坏性操作前都会请求你的确认。Karpathy 在几分钟内从零构建了一个 React 应用。

Karpathy 使用的底层模型是 Claude 3.7 Sonnet。关键认识是：只有当你足够理解模型，能够引导并纠正它时，这类工具才最强大；不能只是输入提示词然后碰运气。

第 9 章 · 多模态

语音与
音频

Karpathy 大约有 一半查询通过语音完成 ，使用的是 Super Whisper——这是他在 Super Whisper、WhisperFlow 和 MacWhisper 中的选择。按下快捷键，说话，再按一次——查询就会被转写并发送。无需打字，摩擦很低。

ChatGPT 的 高级语音模式 走得更远：音频 token 直接进出模型，中间没有文本转写层。因此体验更像真正的对话，而不是套了一层文字转语音。

NotebookLM （Google）可以从你的文档生成音频播客。上传论文、书籍或笔记后，它会生成双主持人讨论。Karpathy 会在散步和长途驾驶时使用它，被动学习自己专业之外的主题。

Karpathy 的语音使用建议 如果查询包含产品名、库名或技术术语，请改用打字。Whisper 经常把小众技术词汇转写错。语音最适合自然语言问题。

第 10 章 · 视觉输入

视觉与
摄像头

现代 LLM 可以接收图像输入——照片、截图、扫描件、图表。模型能够像处理文本一样流畅地推理视觉内容，因为训练数据中包含数十亿级图文对。

Karpathy 的示例包括：上传一张 血液检测扫描件 让模型解读；把摄像头对准一台 Aeronet 4 二氧化碳监测仪 让模型识别设备并解读 713 PPM 读数；以及展示一张 《指环王》地图 ，模型正确识别为中土世界。

视觉能力在资料充分的对象上最可靠——例如血检参考范围、常见消费设备、著名地图等，因为训练数据充分覆盖这些领域。对于专有或罕见对象，更容易出现幻觉。

适合视觉能力的场景 识别未知物体、解读标准实验室检查结果、解释图表和示意图、对印刷文本做 OCR、阅读手写内容、分析截图。

第 11 章 · 个性化

记忆与
个性化

默认情况下，每次对话都是无状态的——关闭标签页后，模型会忘记一切。有两个功能会改变这一点： 记忆 （ChatGPT 会跨会话自动保存与你有关的事实）和 自定义指令 （一个会影响每次回答的持久系统提示词）。

Karpathy 的自定义指令包括：要求尽量采用教育性表达（“能讲清楚就尽量讲清楚”）、为韩语学习设定正式语体，并分享自己的工作和兴趣背景。

可以把自定义指令看作你的个人系统提示词——它会在每次对话前加载。好的指令能压缩那些你本来每次都要重复的偏好，让每次会话都像已经了解你一样。

入门版自定义指令 “保持简洁。当代码和文字都能解决问题时，优先给代码。收到文档时，先用一段话总结。明确标注你的假设。我从事 [你的领域]。”
自定义指令 · ChatGPT
ChatGPT 应该了解你的哪些信息？
我是一名对机器学习感兴趣的软件工程师。我偏好简洁、技术性的回答。我正在学习韩语——提供韩语文本时，默认使用礼貌正式语体（합쇼체）。
ChatGPT 应该如何回答？
解释概念时要有教学性。先给最重要的信息。多使用代码片段。明确标注你做出的任何假设。

第 13 章 · 总结

关键
要点

01
你正在和一个 ZIP 文件对话
模型把互联网压缩进权重中。知识大约滞后 6–12 个月，输出是概率性的，并且在上下文窗口之外没有工作记忆。它无法自行验证自己的答案。
基础
02
了解你的套餐层级和模型
免费版 → 受限。20 美元/月 → GPT-4o / Claude Sonnet。200 美元/月 → o1 Pro、深度研究。让模型匹配任务——困难推理用思考模型，简单查询用快速模型。
模型
03
只为时效性信息使用搜索
对于长期稳定、记录充分的知识，模型权重已经足够，可以跳过搜索。对于近期事件、变化中的情况或小众主题，应启用搜索或使用 Perplexity。
搜索
04
用深度研究做多来源综合
5–15 分钟，20–30 个来源，结构化报告。对文献综述和尽职调查确实有用。ChatGPT 当前需要 200 美元/月套餐；Perplexity 更便宜。
研究
05
核验代码与数据输出
高级数据分析运行的是真实 Python——但模型可能在自己写的代码中幻觉出数值。要用源数据核对数字，而不只是看图表形状。
代码
06
语音减少一半输入摩擦
基于 Whisper 的听写工具消除了打字障碍。Karpathy 大约 50% 的查询通过语音完成。遇到 Whisper 容易转写错误的技术产品名和库名时，使用文本输入。
语音
07
ChatGPT 目前仍是默认选择
功能最多、生态最大、用户体验最成熟。编程用 Claude，搜索优先用 Perplexity。格局变化很快，决定前先查看 LM Arena 的最新排名。
生态系统
